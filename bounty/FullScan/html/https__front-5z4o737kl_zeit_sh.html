    Frequently Asked Questions

    What is serverless deployment? How is it different?

    Serverless is a widely used and debated term. For us, it boils down to:

    • No servers to manage, configure, scale, orchestrate, operate and monitor
    • No server entities or abstractions in your code.

    In practice, this means that all you have to worry about is the code that executes as a result of an incoming HTTP request or event.

    This is not unlike how the popular and well-proven model of writing .php or .pl files and putting them in a webroot works.

    How is ZEIT Now different from other solutions?

    Now makes serverless useful to you and your team directly with tools and workflows that make the underlying cloud infrastructure useful, productive and configuration-free.

    In practical terms, this means:

    • Now provides the framework to avoid re-writing and re-learning everything from scratch to take advantage of serverless today
    • You can deploy popular client-side frameworks (like Next.js, create-react-app, Vue), Node.js or Go APIs as a monorepo with nearly zero configuration
    • We integrate directly with GitHub to deploy upon every push

    What can I deploy to ZEIT Now?

    Any technology that can be served over HTTP and distributed through our CDN network.

    This includes:

    • Static websites and static generators (React, Vue, Angular, etc)
    • Code that renders HTML on the server-side
    • API endpoints that query databases or web APIs and return dynamic data

    In general, most popular technologies and languages already have examples you can start taking advantage of today.

    Is ZEIT Now going to lock me in?

    Now is designed around the exact opposite philosophy.

    There are two parts to a Now deployment:

    • The source code of your application
    • The now.json configuration file which usually describes how to build it

    For the former, one of the crucial decisions we've made is that you shouldn't need to learn a new framework or tool just to use serverless.

    This is what a Node.js API inside a repository hosted on Now looks like:

    module.exports = (req, res) => {
  res.end('Hello World')
}

    The code is fully compliant with the API defined by the Node.js language. Now is able to deploy it to any cloud infrastructure provider by embracing this.

    That's where the builders specified in now.json come in. They are lightweight and fully open source npm modules that compile your code to become universal.

    Is ZEIT Now more expensive than configuring the cloud ourselves?

    No, across the board:

    • Now is fully on-demand and offers prices that average all infrastructure providers
    • For students, open-source developers or hobby projects, we offer a generous free tier
    • No time is spent configuring or setting up complex low-level pipelines

    Our pricing model is designed to grow with you.
